# Internacionalização (i18n) Nativa

Um mapa mundial da história não deve estar restrito apenas a leitores de inglês. O ecossistema CodeOrigins possui um dicionário embutido assíncrono para garantir acesso ao "Big Data" de descrições da história do software no maior número de línguas possivel.

## Como o `localeLoader.ts` Opera

Diferente de sistemas como `react-i18next` que sobrecarregam a aplicação com dicionários pré-cacheados de milhares de chaves, optamos por um fetcher (motor de busca assíncrona) nativo extremamente polido:

A carga de dicionários é feita **dinamicamente** somente quando o Planeta é selecionado e a tela vai abrir, ou no momento zero de boot para elementos de HUD:

1. A pasta `src/lang/` possui dicionários JSON segmentados (Ex: `pt.json`, `en.json`, `es.json`).
2. Tais dicionários possuem sub-chaves como `hud`, `settings`, mas a imensa maioria do volume encontra-se no bloco das linguagens em si (`languages.python.description`).
3. O serviço base usa TypeScript genérico para bater contra essas matrizes JSON em Runtime via Webpack/Vite dinâmico imports: `import("./lang/${locale}.json")`.

> [!TIP]
> Isso mantém o Peso Base do pacote inicial (`bundle.js`) mínimo, puxando as traduções via Rede APENAS de forma tardia (lazy loading) e alocando-as no estado Zustand quando requisitado pela primeira vez.

## Traduzindo Novas Peças de História

Para auxiliar a compor descrições das linguagens, um contribuinte global em CodeOrigins necessitará apenas de seguir o objeto padronizado do planeta dentro de seu respectivo pacote de línguas. O sistema é passivo a novas adições.

```json
// pt.json
{
  "languages": {
    "meu-novo-planeta": {
      "name": "Meu Novo Planeta",
      "description": "Linguagem x criada para y nos anos Z...",
      "paradigm": "Orientado a Protótipos"
    }
  }
}
```

Caso uma chave solicitada não seja encontrada no `pt.json` de um viajonauta brasileiro, o sistema automaticamente fará *fallback* e renderizará o bloco existente no idioma em Inglês (`en`) subjacente para que os botões ou detalhes nunca apresentem bugs brutos de "String Not Found".
